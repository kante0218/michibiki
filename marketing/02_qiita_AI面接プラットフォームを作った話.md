## なぜ作ったのか

転職活動で最も不安なのは「面接」です。

書類は何度でも推敲できます。ポートフォリオは時間をかけて磨けます。しかし面接だけは、**本番と同じ緊張感で練習する手段がほとんどありません。**

- 友人に頼むと遠慮が入り、本音のフィードバックが得られない
- 有料のコーチングは1回数万円。気軽に繰り返せない
- 技術面接に至っては、適切な質問を出せる相手を見つけること自体が難しい

この「練習のボトルネック」をAIで解消できないか——それが導（みちびき）の出発点でした。

## 面接フローの設計

導の面接は3つのフェーズで構成されています。

### Phase 1: 分野選択

33分野（ソフトウェアエンジニアリング、データサイエンス、プロダクトマネジメント、UXデザインなど）から選択。各分野には、その領域の専門家が実際の面接で問う内容を反映したプロンプトテンプレートを用意しています。

### Phase 2: 技術テスト（10問）

選択問題5問＋記述問題5問を、GPT-4で**毎回動的に生成**します。

```typescript
// 問題生成の簡略化したイメージ
const generateQuestions = async (category: string, difficulty: string) => {
  const prompt = `
    ${category}の${difficulty}レベルの技術テストを生成してください。
    - 選択問題5問（4択、正解は1つ）
    - 記述問題5問（2-3文で回答する形式）
    制約: 前回と同一の問題を出さないこと
  `;
  // ...
};
```

**なぜ動的生成にしたか**: 固定の問題プールでは、繰り返し受験した際に問題を暗記できてしまい、実力測定として機能しなくなるためです。

### Phase 3: AIビデオ面接（適応型）

ここが導の核心です。テスト結果をコンテキストとして渡し、**候補者の弱点を深掘りする質問を動的に生成**します。

```
候補者のテスト結果:
- 選択問題: 3/5正解（データ構造の問題で2問不正解）
- 記述問題: アルゴリズムの計算量分析が不十分

→ AIの面接戦略:
  1. まずデータ構造の理解度を確認する質問
  2. 回答の深さに応じて、実務での応用を問う
  3. 計算量分析について具体例で説明を求める
```

従来のAI面接が「テンプレートの質問を順番に投げるだけ」だったのに対し、導では**候補者の回答に応じて次の質問を変える適応型面接**を実現しています。

## SEO設計 — 日本語サービスをどうインデックスさせるか

新サービスのSEOは「鶏と卵」問題です。コンテンツがなければ流入がなく、流入がなければGoogleの評価も上がりません。

導では以下の多層的なアプローチを取りました。

### 1. 構造化データの積極活用

```json
// WebSite, Organization, WebApplication, FAQPage の4つを設定
{
  "@type": "FAQPage",
  "mainEntity": [
    { "name": "AI面接練習はどのように行われますか？", ... },
    { "name": "導（みちびき）は無料で使えますか？", ... },
    // ... 計10問
  ]
}
```

FAQリッチリザルトはクリック率を大幅に向上させます。

### 2. キーワード別ランディングページ

「AI面接 無料」「エンジニア転職」「リモートワーク 求人」など、検索ボリュームの高いキーワードごとに専用ページを作成。各ページにはそのキーワードに特化したmetadata、見出し、コンテンツを配置しています。

### 3. 内部リンクの設計

フッターを4カラム構成にし、「求職者向け」「企業向け」「サポート」「法的情報」で整理。すべてのページから主要ページへの導線を確保しています。

## 得られた学び

### 1. AIの「適応型」は差別化になる

固定テンプレートのAI面接は世の中にすでにあります。しかし、候補者の回答に応じて質問を変える適応型は、体験の質が段違いでした。ユーザーから「本物の面接みたいだった」というフィードバックが最も多く寄せられたポイントです。

### 2. 日本語のSEOは構造化データが効く

日本語圏のWebサービスは、構造化データの実装が遅れている傾向があります。FAQPage、WebSiteスキーマを適切に実装するだけで、競合との差別化ができました。

### 3. 「無料」は最強のグロースレバー

面接練習という領域では、無料であること自体が口コミを生みます。有料化のプレッシャーはありますが、まずはユーザーベースの拡大を優先しています。
